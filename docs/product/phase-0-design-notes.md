# Phase 0 — Product Design Notes

## Problem

New-hire onboarding is fragmented across docs, chats, meetings, and tribal knowledge. Time-to-productivity is slow and inconsistent.

## Principle

LaunchPad must always show the employee:

- what to do now
- why it matters
- who can help
- where the resource is
- how success is measured
- what happens next

## Surfaces (Phase 0 focus)

| Surface | Phase 0 goal |
| --- | --- |
| Marketing web | Brand-led public site shell with trial/demo CTAs |
| Organization admin | Auth + org profile foundation only |
| Employee web | Stub app shell |
| Platform admin | Stub app shell |
| API | Auth, organizations, audit |

## First journeys to design next

1. Signup → create organization → invite admin
2. Invite employee → accept → land on day-1 journey home
3. Manager approves a practical task

Wireframes and detailed user stories live alongside this note as Phase 0 continues.
